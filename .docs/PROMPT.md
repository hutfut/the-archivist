Your task is to build a simple version of NotebookLM. A user should be able to perform the following tasks:

    Upload documents (no preference on file format)
    Delete documents
    Ask questions in a chat experience that an agent answers based on the document content

Your solution only needs to run locally and does not need to be deployed to a cloud. You are welcome to use a mock LLM implementation if you do not have the means to run a real LLM. The Langchain SDK provides a mock LLM implementation, but you may use any implementation you like. There is no expectation that you would pay for tokens to complete the exercise. We will be judging your submission based on innovation, design, and implementation — not the quality of answers from the LLM.

Please prioritize the following stack:

    React for UI
    Node.js or Python for server-side components
    Langchain/Langgraph for agent orchestration

This is an open-ended exercise, so be creative and have fun with it. This is an opportunity to demonstrate what’s important to you and how you like to work. If you complete the basic requirements, show us how creative you can be with your own additions.

You are encouraged to use any tools or libraries that you would normally use on the job, including AI assistants or IDEs. However, please be prepared to explain your work or make adjustments to your solution in real-time during a follow-on discussion. We also encourage you to save your LLM/Agent chat history and submit it with your submission, so we can see your process and how you use these tools, if you do**.**

You may either zip up your repository and email it or create a private GitHub repository and share it when you are ready to submit your work.
